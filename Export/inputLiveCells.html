%inputLiveCells Function
 
%Approach:
 
%Got a make sure that the click stays in the dimensions of the figure
%Prompt the user to alive the cells
%If the user selects, the box will filled black
%Simultaneously the row and column of the box click is being save on an
%array that also stores the previous box clicks
%The textual prompt reproduces itself after each box click
%If the user clicks enter , the textual prompt disappears
%Finally the function returns the ith rows jth columns of the black boxes
%respectively.
 
%use [x,y]=ginput(1) -> this allows the user to select x,y coordinates until
%enter is pressed.
 
function [R, C] = inputLiveCells(m,n)
%Not sure if the figure window is already open or I have to call it
%(m,n) Dimension of the meshgrid
% Initializing output rows and columns
R = [];
C = [];
% Set pointer to a simple arrow
 
while true
[x, y] = ginput(1); % Get a single mouse click
%If nothing is selected then indices will be empty.
col = floor(x); % Column index
row = floor(y); % Row index
if isempty(x) % If "ENTER" is pressed, The function stops.
break;
% Determine the grid cell clicked
% Validate the selected cell is within bounds
elseif row >= 0 && row <= m-1 && col >= 0 && col <= n-1
% Appending to R and C separately
R = [R, row];
C = [C, col];
% Fill the selected cell with black
%fill(row, col, 'k');
set(gcf, 'Pointer', 'arrow')
rectangle('Position', [col, row, 1, 1], 'FaceColor', 'k', 'EdgeColor', [0.5, 0.5, 0.5]);
% 'FaceColor' to fill the rect with 'k' is command for black color
%[1,1] refers to the lenght and width fixing the box.
% if click is out of bounds loop repits/doesnt do anything
%'EdgeColor' and 0.5, color of the rectangle edges.
else
end
end
% Remove the title after the selection is done
title('');
hold off;
end
